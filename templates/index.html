What is Steganography?

Hidden in plain sight

Unlike encryption, which scrambles data so it looks like noise, steganography hides the very existence of a secret. Your message is invisible — the 3D model looks completely normal.

This app encodes your text into the vertex color data of a 3D .OBJ model using LSB (Least Significant Bit) encoding. Only the last bit of each color channel is changed — a difference the human eye cannot detect and most software ignores entirely.

For maximum security, combine steganography with AES-256 encryption — so even if someone discovers a message exists, they still can't read it without the key.

Vertex color before encoding:
R: 1101 1010 G: 0110 1110 B: 1010 0100
Vertex color after encoding 'A' (01000001):
R: 1101 1010 G: 0110 1111 B: 1010 0100
↑ Only 1 bit changed per channel. Visually identical.
Color difference: < 0.4% — below human perception threshold
How It Works

Three simple steps

From message to invisible secret in seconds.

🔒

1 · Hide

Type your secret message, choose a 3D model, and set a password. The message is encoded invisibly into the model's vertex color data using LSB steganography. Optionally enable AES-256 encryption for an extra layer of security and plausible deniability.

Start Encoding →
📤

2 · Share

Two QR codes are generated automatically. QR1 (public) links to the encoded model file — share it openly. QR2 (private) encodes your password key — share only with the intended receiver through a separate, secure channel.

📂 🔑
🔓

3 · Reveal

The receiver scans both QR codes, uploads the model file, and enters the password key. The hidden message is extracted and displayed — with an integrity check confirming the file hasn't been tampered with.
